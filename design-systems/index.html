What we learned #1

After maintaining this for a little while we decided to take our learnings and build something new

Modules

We decide what is defined as a module and build those in isolation.

Version each module independently

This enables everyone to pick and chose what they need and greater transparency into what has changed.

Open source

At Westpac we have many products that are secure and sensitive. Each of those products are responsible for their own implementation and that includes the front end. Downloading minified CSS and and js is hard to asses and put your name on it if you donâ€™t know how it was built. It was important to keep ourself honest and show everyone what we build so project can make their own assessments.

Clear upgrade paths

With modular versioned modules, everyone can chose to upgrade module by module at the pace the projects time and financial commitment allows.

What we built then

And so we built the GUI framework and the blender.

Everythingâ€™s open source

Everything was developed in the open and we had a good team working together to get it done.

Each module is a git submodule

    • Each module is built/tested in isolation
    • Clean git history for each module
    • Coding without merge conflicts
    • One momorepo that manages them all
    • Staging gh-pages per module
Git submodules are not popular but worked great for us. We created tooling to make it even easier.

Core modules hold reusable code

We created five core modules that were included in every blend. javascript, grid, colors, text-styling and fonts

The blender

  • Build your assets online and download a zip
  • Each zip file comes with a "version string"
  • Return to your version anytime
  • Funky add-ons for engagement and smiles
The blender would compile the assets in memory and sent you a zip file. It would generate a version string for you so you can always return to your blend. To make the interaction with the tool more fun, we created funky timely options.

Change-logs FTW

As we created changelogs for each version and we tried to rapidly release new version we had a lot of changelogs to communicate. Slack inspired us to be more entertaining with them. It actually made a very noticeable difference in who and how people engaged with us.

What we learned #2

I have now moved on to the Australian Government and am asked to build a design system here.

5 core modules are 4 too many

It makes the dependency graph unnecessarily complicated.

Nothing is ever small enough

Our icon groups were too big with 6 icons.

Dependencies need more juice

Interdependencies have not been solved and it meant we started building either bigger modules or repeated ourself.

FOSS needs marketing

Open source is all about marketing. The greatest projects have died because they weren't shiny enough.

What weâ€™re building now

At the DTA we are building the new uikit.

cli first

Building cli first now means we can progressively enhance the experience.

One core only

One core, simple, straight forward, less assumptions.

No output for core

Size impact at an absolute minimum.

Monorepo

We are trying out lerna but are still early in the process. Not because submodules are bad, just to learn something new again.

npm modules

Delivery and dependencies solved. Now we just need to make sure yarn is supported.

Flavors of modules

With interdependencies out of the way we can build flavors of modules.

Support for react, vue etc

Templates should enable us to scale but we start with vanilla.

Presets

Npm comes with presets built in. Like babel we can now have a set for e.g. annual reports.

  • Awesome mixins, great error messages 
    
/**
 * Return a space value based on 4 and the unit em
 *
 * @param  {number} $number - The space as a multiplier of 4
 * @return  {number}               - The space in em's
 */
@function uikit-space( $number ) {
	@if type-of( $number ) != 'number' {
		@error "Captain I'm giving it all she's got but: the uikit-space mixin only takes a number!";
	}

	@if type-of( $number ) == 'number' and not unitless( $number ) {
		$number: $number / ($number * 0 + 1);
	}

	$space: $number * 4;

	@return #{ $space }em;
}
